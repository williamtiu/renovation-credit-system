# 🛠️ DecoFinance 系統開發與呈現分工表 (Team Roles & Responsibilities)

## 📌 項目背景與分工概述 (Project Context & Delegation Overview)
基於 DecoFinance 裝修信貸系統的雙 UI 架構（Flask Jinja & React）、多角色權限模型（Customer, Company, Reviewer, Admin）、合規信任評分機制與自動化測試需求，本團隊分工旨在確保**商業邏輯、核心算法、架構安全、前端體驗及產品質量**均能達到 FinTech 行業標準以順利完成 Demo 及交付。

## 🎯 深度任務細分與系統對齊 (Deep Task Analysis & Alignment)

### 1. @Laurance 林 (Product Manager / 產品經理)
**系統職責對齊：**
* **業務工作流設計：** 定義 DecoFinance 中最核心的使用者旅程（如：客戶發布項目 -> 裝修公司競標 -> 智能合約簽署 -> 里程碑驗收與資金釋放）。
* **市場缺口與商業驗證：** 針對香港裝修業界的痛點（爛尾樓、裝修公司融資難），撰寫並完善業務方案 (`14_Executive_Summary.md` & 產業數據分析)。
* **Demonstration 統籌：** 負責最終 PPT 視覺設計、Demo Script (`06_Presentation_Script_ch.md`) 的編寫，確保演示過程流暢且涵蓋所有亮點（如信用報告匯出、糾紛處理）。
**📦 關鍵交付物：** 市場分析與商業建議書、高質量演示 PPT 展示、操作流程腳本 (Workflow Examples)。

### 2. @成友 (Backend Engineer / 後端工程師)
**系統職責對齊：**
* **核心算法實作 (Credit Scorer)：** 負責實作包含財務、營運、資質與客評的「四維度信用評分引擎」(`services/credit_scorer.py`)。
* **API 與服務層開發：** 建立穩定且高效的 RESTful API (`routes/api.py`)，推動貸款申請、審批邏輯與資金狀態流轉 (`loans.py`, `projects.py`)。
* **智能合約與數據處理：** 構建 Escrow (第三方託管) 的模擬邏輯，確保付款節點的資金凍結/釋放機制的技術正確性。
**📦 關鍵交付物：** Flask 後端核心源碼、核心評分引擎算法模組、API 接口。

### 3. @Tracy (DB Engineer & QA / 數據庫工程師與測試)
**系統職責對齊：**
* **數據建模與 Schema 維護：** 基於 SQLAlchemy 設計關聯式資料庫架構 (`models/` 目錄下的 users, companies, escrow_ledger_entries_cases 等表) (`03_Database_ch.md`)。
* **測試數據種子 (Data Seeding)：** 開發並擴展 `seed_db.py` 與隨機數據生成腳本，確保 Demo 時環境能一鍵構建豐富的測試數據。
* **自動化質量保證 (Selenium & Pytest)：** 負責端到端全鏈路測試 (`test_ui_selenium.py` 以及核心邏輯的 Pytest)，確保表單提交流程無金流與權限漏洞。
**📦 關鍵交付物：** 系統 ERD 數據庫架構圖、測試覆蓋率與自動化測試報告 (`12_Test_Report.md`)、種子數據生成腳本。

### 4. @哥斯拉🦕 (Frontend & UI Designer / 前端 UI 設計師)
**系統職責對齊：**
* **現代化 UI/UX 實現：** 負責 DecoFinance 雙棲 UI 的前端交互開發（傳統 Jinja 模板及基於 React/Vite 的無頭模式 `/new-ui`），確保介面符合金融業嚴謹且具設計感的風格。
* **儀表板設計 (Dashboards)：** 開發針對不同角色的自定義看板（如 Reviewer 的 Dashboard 圖表、Company 的信用分數可視化組件）。
* **UX 體驗優化：** 優化多步驟表單（註冊流程、裝修資質認證文件上傳、PDF 信用報告渲染與匯出功能），縮短操作路徑。
**📦 關鍵交付物：** 前端應用源碼 (Jinja templates & Vite/React src)、系統原型設計/版面視覺規範、圖表與 UI 說明文檔。

### 5. William Tiu (Tech Coordinator & Compliance / 技術統籌與合規架構)
**系統職責對齊：**
* **系統架構整合：** 擔任前後端對接的橋樑，維護 `02_Architecture_ch.md`，定義不同服務藍圖與交互標準。
* **RBAC 權限與安全性防護：** 設計並落實角色的嚴格安全邊界 (`customer`, `company_user`, `reviewer`, `admin`)，確保路由裝飾器攔截未授權訪問（見 `角色流程与权限限制详解.md`）。
* **ESG/OSH 合規指標納入：** 將香港建築署(BD)、勞工處職業安全(OSH)等合規驗證欄位完整映射到資料庫與評分體系。
* **最終文檔與報告統籌：** 負責梳理與校對全專案的 Markdown 技術文檔庫 (`13_Submission_Index.md`)，確保系統術語、邏輯一致性達到交付標準。
**📦 關鍵交付物：** 全域技術架構圖、RBAC 權限控制分析報告、系統安全與合規機制文檔、專案交付 Index 總目錄。
