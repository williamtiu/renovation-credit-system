# 🗄️ Database Design Document

## 1. Database Overview

### 1.1 Database Selection
| Environment | Choice | Justification |
|------|------|------|
| Development | SQLite 3 | Lightweight, zero-config, good for prototyping |
| Production | PostgreSQL 15+ | High performance, Concurrency, ACID |
| ORM | SQLAlchemy 2.0 | Python ecosystem standard, Type safe |

### 1.2 Database Standards
- **Charset**: UTF-8
- **Timezone**: Asia/Hong_Kong (UTC+8)
- **Naming Conventions**: Set entirely in snake_case (e.g. `loan_applications`). PK is `id`, FK is `{table_singular}_id`.

## 3. Data Table Detailed Design

### 3.1 users
Table identifying all user login components
- `id` (INTEGER, PK): Internal ID
- `email` (VARCHAR(100), Unique): Email address / Login
- `password_hash` (VARCHAR(255)): Bcrypt Hash
- `role` (VARCHAR(20)): 'company', 'admin', 'bank'
- `company_id` (FK): Links to `companies`
- `bank_id` (FK): Links to `banks`

### 3.2 companies
Defines details for renovation company registrants
- `id` (INTEGER, PK): Primary Key
- `company_name` (VARCHAR(200))
- `business_registration` (VARCHAR(50), Unique): BR Number (8 digits)
- `established_date` (DATE)
- `annual_revenue` (DECIMAL)
- `employee_count` (INTEGER)
- `status` (VARCHAR): active, suspended
- `risk_level` (VARCHAR): low, medium, high
- `has_license` (BOOLEAN)

### 3.3 credit_scores
The periodic or manual score assessments generated by the algorithm.
- `id` (INTEGER, PK)
- `company_id` (INTEGER, FK)
- `credit_score` (INTEGER): Rating out of 1000
- `credit_grade` (VARCHAR): AAA, AA, A, BBB, BB, B, C
- Sections include `financial_strength_score`, `operational_stability_score`, `credit_history_score`, `qualification_score`, `industry_risk_score`.
- `recommended_loan_limit`, `recommended_interest_rate`

### 3.4 loan_applications
- `loan_amount` (DECIMAL): Requested base HKD sum
- `application_status` (VARCHAR): pending, under_review, approved, rejected, withdrawn, disbursed, completed
- `approval_date`, `approved_amount`, `approved_interest_rate`
- `company_id` (FK), `bank_id` (FK)

### 3.5 banks
Details about external banking providers participating in the Reno system
- `bank_name`, `contact_person`, `email`, `max_loan_amount`, `status`

### 3.6 audit_logs
Auditing platform for system security compliance & trace-backs.
- `action` (VARCHAR): CREATE, UPDATE, DELETE, LOGIN, LOGOUT
- `table_name` (VARCHAR)
- `old_value`, `new_value` (JSON text arrays)
- `ip_address` (VARCHAR)

## 4. Backup & Migration Strategy

**Backup Intervals:**
- Full Database Backups: Daily (Retained 30 Days)
- Incremental Backups: Hourly (Retained 7 Days)
- Audit Logs: Safely retained for 7 Years strictly to meet compliance.

**Migrations**: Utilizing Python `alembic` commands internally for forward/backward DB revisions.
