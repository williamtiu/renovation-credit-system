## 📊 体系概述

### 核心理念

DecoFinance 的**新4大维度信托评分体系**是一个专为装修行业设计的综合性信用评估系统。该体系基于最新的评分要求，从**财务实力、运营稳定性、资质认证、客户评价**四个维度进行全面评估，为金融机构提供科学、透明的信任度评估。

### 评分范围

- **总分范围**：0 - 1000 分
- **分数越高**：表示公司信任度越高，贷款风险越低
- **评估周期**：每 180 天更新一次
- **数据保密**：所有财务数据仅用于内部计算，不对外公开

## 🏗️ 评分架构

### 计算公式

```
总评分 = 财务实力评分 + 运营稳定性评分 + 资质与认证评分 + 客户评价评分

其中：
- 财务实力评分：150-600 分
- 运营稳定性评分：80-250 分
- 资质与认证评分：0-200 分
- 客户评价评分：60-300 分
```

### 1️⃣ 财务实力评分 (Financial Strength)

**满分**：600 分

**评估指标**：

| 指标 | 计算方式 | Poor | Normal | Good | 得分范围 |
|------|---------|------|--------|------|---------|
| **注册资本** | 公司注册资金 | <50万 | 50-100万 | ≥1000万 | 30-150分 |
| **年营收** | 公司年收入 | <500万 | 500-5000万 | ≥5000万 | 30-150分 |
| **流动比率** | 流动资产/流动负债 | <1.1 | 1.1-1.6 | >1.6 | 50-150分 |
| **现金比率** | 现金总额/总负债 | <1.1 | 1.1-1.6 | >1.6 | 50-150分 |
| **债务权益比** | 总负债/股东权益 | >2 | 1-2 | <1 | 50-150分 |

**详细说明**：

#### 注册资本 (30-150 分)
```python
if company.registered_capital >= 10000000:
    score += 150
elif company.registered_capital >= 5000000:
    score += 120
elif company.registered_capital >= 1000000:
    score += 90
elif company.registered_capital >= 500000:
    score += 60
else:
    score += 30
```

#### 年营收 (30-150 分)
```python
if company.annual_revenue >= 50000000:
    score += 150
elif company.annual_revenue >= 20000000:
    score += 120
elif company.annual_revenue >= 10000000:
    score += 90
elif company.annual_revenue >= 5000000:
    score += 60
else:
    score += 30
```

#### 流动比率 (30-150 分)
```python
current_ratio = company.current_assets / company.current_liabilities
if current_ratio > 1.6:
    score += 150  # Good
elif current_ratio >= 1.1:
    score += 100  # Normal
else:
    score += 50   # Poor
```

#### 现金比率 (30-150 分)
```python
cash_ratio = company.total_cash / company.total_liabilities
if cash_ratio > 1.6:
    score += 150  # Good
elif cash_ratio >= 1.1:
    score += 100  # Normal
else:
    score += 50   # Poor
```

#### 债务权益比 (30-150 分)
```python
debt_to_equity = company.total_liabilities / company.shareholders_equity
if debt_to_equity < 1:
    score += 150  # Good
elif debt_to_equity <= 2:
    score += 100  # Normal
else:
    score += 50   # Poor
```

## 🎯 信用等级评定

### 信用等级划分

| 等级 | 分数范围 | 风险水平 | 贷款利率 | 贷款倍数 |
|------|---------|---------|---------|---------|
| **AAA** | 751 - 1000 | 极低风险 | 3.5% | 2.0x |
| **AA** | 701 - 750 | 低风险 | 4.0% | 1.8x |
| **A** | 651 - 700 | 中低风险 | 4.5% | 1.5x |
| **BBB** | 601 - 650 | 中等风险 | 5.5% | 1.2x |
| **BB** | 551 - 600 | 中高风险 | 6.5% | 0.8x |
| **B** | 501 - 550 | 高风险 | 8.0% | 0.5x |
| **C** | 0 - 500 | 极高风险 | 10.0% | 0.2x |

### 风险等级映射

| 总分 | 风险等级 |
|------|---------|
| ≥ 700 | 低风险 (low) |
| 550 - 699 | 中风险 (medium) |
| < 550 | 高风险 (high) |

### 贷款额度计算

```python
推荐贷款额度 = 年收入 × 信用等级对应倍数

例如：
- AAA 级公司，年收入 1000 万 → 可贷 2000 万
- B 级公司，年收入 1000 万 → 可贷 500 万
```

## ⚠️ 风险因素识别

### 自动识别的风险因素

系统会自动识别并记录以下风险因素：

#### 财务风险
- ❌ 无注册资本信息
- ❌ 无年营收信息
- ❌ 流动比率过低（< 1.1）
- ❌ 现金比率过低（< 1.1）
- ❌ 债务权益比过高（> 2）

#### 运营风险
- ❌ 经营历史短（< 2 年）
- ❌ 项目完成历史有限（< 10 个）
- ❌ 员工基数小（< 10 人）

#### 资质风险
- ❌ 缺少商业注册
- ❌ 缺少小型工程注册
- ❌ 小型工程注册未验证
- ❌ 保险文件未上传
- ❌ 保险未验证
- ❌ 缺少 OSH 安全主任信息
- ❌ OSH 安全主任未验证

#### 客户评价风险
- ❌ 平均客户评分低（< 3.0/5）
- ❌ 主观评估分数低（< 50）

#### 其他风险
- ❌ 公司状态异常（suspended/blacklisted）

### 风险因素输出示例

```json
{
  "risk_factors": [
    "Missing registered capital information",
    "Current ratio too low (0.95)",
    "Short operating history",
    "Minor works registration not verified",
    "Insurance not verified",

    "Low average customer rating (2.8/5)",
    "Company status is suspended"
  ]
}
```

## 📱 应用场景

### 1. 贷款审批决策

**审批流程**：
```
申请提交 → 自动评分 → 风险评级 → 审批决策
   ↓          ↓          ↓          ↓
 资料审核   计算总分   确定等级   批准/拒绝
```

**决策规则**：
- **AAA/AA 级**：自动批准，快速放款
- **A/BBB 级**：人工审核，标准流程
- **BB/B 级**：严格审核，附加条件
- **C 级**：建议拒绝

### 2. 利率定价

根据信用等级自动确定贷款利率：
- AAA 级：3.5%（最优惠）
- C 级：10.0%（风险补偿）

### 3. 贷款额度控制

根据信用等级和年收入自动计算最高额度：
```python
max_loan_amount = annual_revenue * LOAN_MULTIPLIER[credit_grade]
```

### 4. 监控预警

**触发条件**：
- 分数下降超过 50 分
- 信用等级下调
- 新增风险因素
- 合规状态变更

### 5. 公司对比

**对比维度**：
- 总分对比
- 各维度得分对比
- 风险因素数量对比
