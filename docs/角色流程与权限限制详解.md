# DecoFinance 系统角色流程与权限限制详解

**更新时间**: 2026-03-24  
**版本**: v2.0 (最新权限控制)

---

## 一、角色定义与核心职责

### 1. customer（客户/业主）
**职责**: 装修项目发起方，负责发布项目、选择承包商、审批里程碑和付款

**核心权限**:
- ✅ 项目管理（创建、编辑、删除自己的项目）
- ✅ 招标管理（接受或拒绝投标）
- ✅ 里程碑审批
- ✅ 发起争议
- ❌ 不能管理公司
- ❌ 不能处理贷款审批

---

### 2. company_user（公司用户/承包商）
**职责**: 装修公司代表，负责投标、执行项目、申请贷款

**核心权限**:
- ✅ 公司管理（创建**1 个**公司、编辑自己的公司）
- ✅ 项目投标
- ✅ 贷款申请和还款
- ✅ 提交项目里程碑
- ❌ 不能审批贷款
- ❌ 不能解决争议
- ❌ 不能删除数据

**特殊限制**:
- ⚠️ **一对一限制**: 每个公司用户只能创建或关联一个公司
- ⚠️ **数据隔离**: 只能查看自己公司的数据

---

### 3. reviewer（审核员）
**职责**: 平台审核人员，负责贷款审批、争议解决

**核心权限**:
- ✅ 贷款审批（批准/拒绝）
- ✅ 贷款发放
- ✅ 争议解决
- ✅ 查看所有数据（公司、贷款、项目、争议）
- ✅ 审计日志查看
- ❌ 不能删除公司/贷款
- ❌ 不能创建项目或公司

---

### 4. admin（管理员）
**职责**: 系统管理员，拥有最高权限

**核心权限**:
- ✅ 所有角色的全部权限
- ✅ 删除公司、贷款、项目
- ✅ 用户管理
- ✅ 系统配置

---

## 二、详细业务流程与权限控制

### 2.1 公司管理流程

#### 流程 1: 添加公司

```
路径：GET/POST /companies/add
权限：admin, reviewer, company_user
```

**流程步骤**:

1. **访问权限检查**
   - `@role_required('admin', 'reviewer', 'company_user')`
   - 必须登录且为指定角色

2. **公司用户特殊限制**
   ```python
   if g.user.role == 'company_user' and not company_user_can_add(g.user):
       flash('您已经关联了一个公司，不能再添加新的公司。', 'warning')
       return redirect(url_for('companies.list_companies'))
   ```
   - 检查条件：`user.company_id is None`
   - 如果已关联公司，重定向到列表页

3. **创建公司**
   - 填写公司信息表单
   - 自动计算投标资格 (`_derive_bid_eligibility`)
   - 保存到数据库

4. **关联用户**
   ```python
   if g.user.role == 'company_user' and g.user.company_id is None:
       g.user.company_id = company.id
   ```
   - 公司用户创建公司后自动关联
   - 管理员和审核员不会自动关联

**权限矩阵**:

| 角色 | 可访问 | 数量限制 | 自动关联 |
|------|--------|----------|----------|
| admin | ✅ | 无限制 | ❌ |
| reviewer | ✅ | 无限制 | ❌ |
| company_user | ✅ | 仅 1 个 | ✅ |
| customer | ❌ | - | - |

```python
def can_manage_company(user, company):
    if user is None:
        return False
    if user.role == 'admin':
        return True
    if user.role == 'reviewer':
        return True
    return user.company_id == company.id  # 公司用户只能编辑自己的公司
```

#### 流程 4: 查看项目详情

```
路径：GET /projects/<id>

```

**访问控制逻辑**:

```python
def _require_project_access(project):
    if g.user.role in ['admin', 'reviewer']:
        return  # 无限制
    if g.user.role == 'customer':
        require_ownership(project.customer_user_id == g.user.id)
        return
    if g.user.role == 'company_user':
        company = _current_user_company()
        require_ownership(
            project.status == 'open_for_bids' or 
            _company_has_project_access(project, company.id)
        )
        return
    require_ownership(False)
```

**公司访问项目判断**:

```python
def _company_has_project_access(project, company_id):
    if not company_id:
        return False
    # 检查公司是否提交了投标或中标
    if any(bid.company_id == company_id for bid in project.bids):
        return True
    return bool(
        project.accepted_bid_id and 
        any(bid.id == project.accepted_bid_id and bid.company_id == company_id for bid in project.bids)
    )
```

**权限矩阵**:

| 角色 | 可访问 | 条件 |
|------|--------|------|
| customer | ✅ | 自己创建的项目 |
| company_user | ✅ | 开放投标项目 OR 已提交投标 OR 中标项目 |
| reviewer | ✅ | 所有项目 |
| admin | ✅ | 所有项目 |

### 3.1 装饰器

#### `@login_required`
```python
def login_required(f):
    @wraps(f)
    def decorated_function(*args, **kwargs):
        if 'user_id' not in session:
            flash('Please log in to access this page.', 'warning')
            return redirect(url_for('auth.login'))
        if getattr(g, 'user', None) is not None and not g.user.is_active:
            session.pop('user_id', None)
            flash('Your account is inactive.', 'danger')
            return redirect(url_for('auth.login'))
        return f(*args, **kwargs)
    return decorated_function
```

**功能**:
1. 检查用户是否登录
2. 检查账户是否激活
3. 未登录重定向到登录页

---

#### `@role_required(*roles)`
```python
def role_required(*roles):
    def decorator(f):
        @wraps(f)
        @login_required
        def decorated_function(*args, **kwargs):
            if g.user is None or g.user.role not in roles:
                flash('You do not have permission to access this page.', 'danger')
                return redirect(url_for('main.index'))
            return f(*args, **kwargs)
        return decorated_function
    return decorator
```

**功能**:
1. 内置 `@login_required`
2. 检查用户角色是否在允许的角色列表中
3. 无权限重定向到首页

#### 项目访问控制 `_require_project_access(project)`

```python
def _require_project_access(project):
    if g.user.role in ['admin', 'reviewer']:
        return
    if g.user.role == 'customer':
        require_ownership(project.customer_user_id == g.user.id)
        return
    if g.user.role == 'company_user':
        company = _current_user_company()
        require_ownership(
            project.status == 'open_for_bids' or 
            _company_has_project_access(project, company.id if company else None)
        )
        return
    require_ownership(False)
```

#### 公司投标资格检查 `_company_has_project_access(project, company_id)`
```python
def _company_has_project_access(project, company_id):
    if not company_id:
        return False
    # 检查公司是否提交了投标或中标
    if any(bid.company_id == company_id for bid in project.bids):
        return True
    return bool(
        project.accepted_bid_id and 
        any(bid.id == project.accepted_bid_id and bid.company_id == company_id for bid in project.bids)
    )
```

## 七、权限配置最佳实践

### 7.1 权限设计原则

1. **最小权限原则**: 用户仅拥有完成工作所需的最小权限
   - 公司用户：只能访问自己公司的数据
   - 客户：只能管理自己的项目
   - 审核员：只能审批，不能删除

2. **职责分离原则**: 关键操作需要多人完成
   - 贷款申请（公司用户）vs 贷款审批（审核员/管理员）
   - 项目创建（客户）vs 项目投标（公司用户）
   - 里程碑提交（公司用户）vs 里程碑审批（客户）

3. **数据隔离原则**: 敏感数据按角色隔离
   - 公司数据：公司用户仅看自己
   - 贷款数据：公司用户仅看自己
   - 项目数据：按参与关系过滤

4. **审计追踪原则**: 所有关键操作都有日志
   - 创建、更新、删除操作
   - 审批、拒绝操作
   - 资金相关操作

---

### 7.2 安全建议

1. **定期审查高权限账户**
   - reviewer 和 admin 角色权限较大
   - 建议定期审查操作日志
   - 限制 admin 账户数量

2. **监控异常操作**
   - 频繁删除操作
   - 非工作时间的审批操作
   - 大额贷款审批

3. **账户 inactive 处理**
   - 及时禁用离职员工账户
   - 定期清理未激活账户

4. **前后端双重验证**
   - 前端：按钮显示控制（用户体验）
   - 后端：权限验证（安全保障）
   - 不依赖单一验证点

5. **敏感操作二次确认**
   - 删除操作：前端确认对话框
   - 贷款审批：后端状态检查
   - 资金操作：审计日志记录

---

### 7.3 常见问题排查

**问题 1: 公司用户看不到公司列表**
- 检查：`user.company_id` 是否为 None
- 解决：确保用户已关联公司

**问题 2: 公司用户无法编辑公司**
- 检查：`user.company_id == company.id`
- 解决：确认用户关联到正确的公司

**问题 3: 公司用户提交贷款申请失败**
- 检查：公司是否关联到用户
- 检查：项目是否关联到公司（如有）
- 解决：确保公司和项目关系正确

**问题 4: 客户无法看到项目**
- 检查：`project.customer_user_id == user.id`
- 解决：确认项目所有权

**问题 5: 公司用户无法投标**
- 检查：公司验证状态
  - 牌照验证：`licence_verification_status == 'verified'`
  - 保险验证：`insurance_verification_status == 'verified'`
  - 安全培训：`safety_training_coverage >= 80`
- 解决：完成公司验证
