## 📊 体系概述

### 核心理念

DecoFinance 的**合规感知信托评分体系**是一个专为装修行业设计的综合性信用评估系统。该体系不仅评估传统财务指标，更强调**合规性**和**行业特定风险因素**，为金融机构提供全面的信任度评估。

### 评分范围

- **总分范围**：0 - 1000 分
- **分数越高**：表示公司信任度越高，贷款风险越低
- **评估周期**：每 180 天更新一次

### 八大评估因子

```
┌─────────────────────────────────────────────────────────┐
│           DecoFinance 信托评分构成                        │
├─────────────────────────────────────────────────────────┤
│  1. 付款历史 (Payment History)              最高 500 分  │
│  2. 项目完成率 (Project Completion Rate)    最高 400 分  │
│  3. 合规与执照 (Compliance & Licensing)     最高 400 分  │
│  4. 财务稳定性 (Financial Stability)        最高 800 分  │
│  5. 客户满意度 (Customer Satisfaction)      最高 500 分  │
│  6. 保险覆盖 (Insurance Coverage)           最高 300 分  │
│  7. OSH 合规 (OSH Compliance)               最高 300 分  │
│  8. ESG 表现 (ESG Performance)              最高 300 分  │
└─────────────────────────────────────────────────────────┘
```

## 🏗️ 评分架构

### 计算公式

```
总评分 = 财务实力评分 + 运营稳定性评分 + 信用历史评分 + 
        资质评分 + 行业风险评分 + 合规调整分

其中：
- 财务实力评分：150-300 分
- 运营稳定性评分：80-250 分
- 信用历史评分：80-240 分
- 资质评分：0-115 分
- 行业风险评分：30-85 分
- 合规调整分：-100 至 +100 分
```

## ⚖️ 合规感知调整

### 核心特色：合规调整分

**调整范围**：-100 分 至 +100 分

这是本评分体系的**核心创新**，专门针对装修行业的合规要求进行精细化评估。

### 评估维度

#### 1. 执照合规性（±70 分）

| 状态 | 调整分 |
|------|--------|
| 已验证 (verified) | +30 分 |
| 已拒绝 (rejected) | -40 分 |
| 待审核 (pending) | 0 分 |

#### 2. 保险合规性（±40 分）

| 状态 | 调整分 |
|------|--------|
| 已验证 (verified) | +20 分 |
| 已拒绝 (rejected) | -20 分 |
| 待审核 (pending) | 0 分 |

#### 3. 纠纷历史（最高 -40 分）

```python
adjustment -= min(40, dispute_count_cached * 10)
```

每个纠纷扣 10 分，最多扣 40 分

#### 4. 职业安全健康（OSH）合规（±35 分）

| 指标 | 条件 | 调整分 |
|------|------|--------|
| **OSH 政策** | 有政策 | +20 分 |
| | 无政策 | -15 分 |
| **安全培训覆盖率** | ≥ 90% | +20 分 |
| | ≥ 80% | +10 分 |
| | 60-79% | -5 分 |
| | < 60% | -20 分 |

#### 5. 重物搬运合规（±20 分）

| 指标 | 条件 | 调整分 |
|------|------|--------|
| **16kg 搬运合规** | 已确认 | +10 分 |
| | 未确认 | -10 分 |
| **起重设备可用** | 有设备 | +5 分 |
| | 无设备 | -5 分 |

#### 6. 安全事故记录（最高 -30 分）

```python
adjustment -= min(30, safety_incident_count * 10)
```

每个安全事故扣 10 分，最多扣 30 分

#### 7. ESG 表现（±20 分）

| ESG 政策级别 | 调整分 |
|------------|--------|
| Advanced（高级） | +15 分 |
| Basic（基础） | +8 分 |
| None（无） | -5 分 |

#### 8. 绿色材料使用（±10 分）

| 绿色材料比例 | 调整分 |
|------------|--------|
| ≥ 40% | +10 分 |
| ≥ 20% | +5 分 |
| < 20% | 0 分 |

## 🎯 信用等级评定

### 信用等级划分

| 等级 | 分数范围 | 风险水平 | 贷款利率 | 贷款倍数 |
|------|---------|---------|---------|---------|
| **AAA** | 751 - 1000 | 极低风险 | 3.5% | 2.0x |
| **AA** | 701 - 750 | 低风险 | 4.0% | 1.8x |
| **A** | 651 - 700 | 中低风险 | 4.5% | 1.5x |
| **BBB** | 601 - 650 | 中等风险 | 5.5% | 1.2x |
| **BB** | 551 - 600 | 中高风险 | 6.5% | 0.8x |
| **B** | 501 - 550 | 高风险 | 8.0% | 0.5x |
| **C** | 0 - 500 | 极高风险 | 10.0% | 0.2x |

### 风险等级映射

| 总分 | 风险等级 |
|------|---------|
| ≥ 700 | 低风险 (low) |
| 550 - 699 | 中风险 (medium) |
| < 550 | 高风险 (high) |

### 贷款额度计算

```python
推荐贷款额度 = 年收入 × 信用等级对应倍数

例如：
- AAA 级公司，年收入 1000 万 → 可贷 2000 万
- B 级公司，年收入 1000 万 → 可贷 500 万
```

## ⚠️ 风险因素识别

### 自动识别的风险因素

系统会自动识别并记录以下风险因素：

#### 财务风险
- ❌ 无执照信息
- ❌ 经营历史短（< 2 年）
- ❌ 高债务收入比（> 70%）
- ❌ 还款历史差
- ❌ 年收入低（< 100 万）

#### 合规风险
- ❌ 执照验证未完成
- ❌ 保险验证未完成
- ❌ OSH 政策证据缺失
- ❌ 安全培训覆盖率不足（< 80%）
- ❌ 16kg 搬运控制未确认
- ❌ 起重设备可用性未确认

#### 安全风险
- ❌ 安全事故记录（过去 12 个月）

#### ESG 风险
- ❌ ESG 治理框架未声明
- ❌ 绿色材料采用率低（< 20%）

#### 运营风险
- ❌ 纠纷历史记录
- ❌ 公司状态异常（suspended/blacklisted）

### 风险因素输出示例

```json
{
  "risk_factors": [
    "Missing contractor licence information",
    "Short operating history",
    "High debt-to-revenue ratio",
    "Licence verification incomplete",
    "OSH policy evidence missing",
    "Safety training coverage below target (75%)",
    "16kg handling control not confirmed",
    "ESG governance framework not declared",
    "Open dispute history count: 2"
  ]
}
```

## 📱 应用场景

### 1. 贷款审批决策

**审批流程**：
```
申请提交 → 自动评分 → 风险评级 → 审批决策
   ↓          ↓          ↓          ↓
 资料审核   计算总分   确定等级   批准/拒绝
```

**决策规则**：
- **AAA/AA 级**：自动批准，快速放款
- **A/BBB 级**：人工审核，标准流程
- **BB/B 级**：严格审核，附加条件
- **C 级**：建议拒绝

### 2. 利率定价

根据信用等级自动确定贷款利率：
- AAA 级：3.5%（最优惠）
- C 级：10.0%（风险补偿）

### 3. 贷款额度控制

根据信用等级和年收入自动计算最高额度：
```python
max_loan_amount = annual_revenue * LOAN_MULTIPLIER[credit_grade]
```

### 4. 监控预警

**触发条件**：
- 分数下降超过 50 分
- 信用等级下调
- 新增风险因素
- 合规状态变更

### 5. 公司对比

**对比维度**：
- 总分对比
- 各维度得分对比
- 合规调整分对比
- 风险因素数量对比

## 🔧 技术实现

### 数据模型

**CreditScore 表结构**：
```python
class CreditScore(db.Model):
    id                          # 主键
    company_id                  # 公司 ID（外键）
    credit_score                # 总分 (0-1000)
    credit_grade                # 信用等级 (AAA-C)
    financial_strength_score    # 财务实力分
    operational_stability_score # 运营稳定分
    credit_history_score        # 信用历史分
    qualification_score         # 资质分
    industry_risk_score         # 行业风险分
    risk_level                  # 风险等级
    risk_factors                # 风险因素（JSON）
    recommended_loan_limit      # 推荐贷款额度
    recommended_interest_rate   # 推荐利率
    scoring_model_version       # 模型版本
    scored_at                   # 评分时间
    expires_at                  # 过期时间
```

### 评分流程

```python
from services.credit_scorer import CreditScorer

# 1. 创建评分器
scorer = CreditScorer()

# 2. 计算评分
result = scorer.calculate_score(company)

# 3. 保存结果
credit_score = scorer.save_score(company, result, notes="定期评估")
db.session.add(credit_score)
db.session.commit()
```
