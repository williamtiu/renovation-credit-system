# DecoFinance 演示文稿脚本

## 开场
"今天我们将展示 DecoFinance，这是一个专为香港装修行业设计的装修信托和项目融资平台。该平台帮助公司证明可信度，帮助客户控制交付风险，并帮助审查员利用更好的运营数据做出融资决策。"

建议视觉：
- 标题幻灯片后立即显示仪表板或登录屏幕。

## 问题
"通用贷款审查错过了装修中重要的运营信号：牌照有效性、保险状态、职业安全健康准备情况、纠纷历史和里程碑交付绩效。"

## 解决方案
"DecoFinance 在一个系统中结合了信托评分、公司信用报告、项目投标、里程碑治理、内部智能合约逻辑和贷款审查。"

## 角色
"系统支持四种角色：公司用户、客户、审查员和管理员。每个角色看到与其职责相关的工作流。"

## 信托评分和报告
"信托评分保持 1000 分结构，但使其适应装修风险。报告页面将该评分转化为对贷款机构和审查员有用的内容，包括风险因素、验证检查、审计活动和 PDF 导出。"

建议视觉：
- 首先显示公司信用报告页面；
- 然后切换到比较工作区或 PDF 下载操作。

## 仪表板
"仪表板现在显示投资组合可见性，而不仅仅是计数。它包括已验证公司、未决纠纷、已批准贷款量、安全审查待办事项、观察名单公司和趋势图表。"

建议视觉：
- 首先强调投资组合卡片；
- 然后指向评分趋势和纠纷趋势图表。

## 项目和智能合约工作流
"客户可以创建项目并接收来自已验证承包商的投标。一旦接受投标，系统就会激活智能合约协议记录。然后规划、提交和批准里程碑。如果开启纠纷，合约将冻结直到解决。"

建议视觉：
- 显示项目详情页面；
- 指向投标、里程碑和智能合约摘要卡片。

## 贷款工作流
"贷款申请可以选择链接到项目。审查员或管理员用户可以批准或拒绝申请，然后继续通过拨款和还款跟踪。"

## 技术架构
"当前实现是带有 Jinja 模板和 SQLAlchemy 模型的 Flask 单体。文档现在与此仓库中实际运行的代码库匹配。"

## 测试和演示支持
"该项目包括后端流的 pytest 覆盖和 UI 流的 Selenium 覆盖。最新完整回归套件通过 33 个测试。演示数据可以通过固定种子脚本或新的随机数据生成器加载。"

建议视觉：
- 显示一个测试结果日志行；
- 然后显示随机数据生成器命令或批处理提示。

## 闭幕
"DecoFinance 展示了装修特定信托评分、项目控制和融资工作流如何在一个实用、可测试的平台中结合在一起。"
